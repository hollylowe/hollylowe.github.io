Access the code for the DrillAlert Backend:

The DrillAlert application is built on ASP .NET Web API and MVC, and is deployed using Microsoft Azure. The deployment consists of an Azure Web App and an Azure MSSQL Database.

Github Repository

Email mloewent@calpoly.edu with your github profile name and you will be added to the private repository.

Web App
  1. Login to Azure.com web portal
  2. Select new -> compute -> web app -> quick create
  3. Name your web app accordingly (we chose DrillAlert.azurewebsites.net)
  4. After creation, from the dashboard, “download the publish profile”
  5. Open the CapstoneBackend.sln in Visual Studio
  6. In the DrillAlertBackend project, select Web.config
  7. If needed, update the app setting FederationMetadataLocation with a different endpoint
  8. Right-click on the “DrillAlertBackend” -> “Publish…”
  9. Select “Import” from the profile page, and select your publish profile you downloaded from Azure.com
  10. Press “Publish”
  11. Your web application is now deployed
MSSQL Database
  1. Login to Azure.com web portal
  2. Select new -> data services -> SQL database
  3. Name it accordingly and create a new server if necessary (default settings were used)
  4. From the dashboard page, select “Show connection strings”
  5. Take note of the “server”, “username”, and “password” fields
  6. Open the CapstoneBackend.sln in Visual Studio
  7. In each project (DrillAlertBackend, DrillAlertDaemon, and DrillAlertData) replace the existing connection strings in the web.config files
  8. Back on the Azure.com MSSQL Dashboard, select “Open in Visual Studio”
  9. Back in Visual Studio, in the Solution Items folder, run the “NewInit.sql” file to initialize all tables in the database
  10. Your database is now connected and initialized
